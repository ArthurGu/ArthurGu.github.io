HGAME 2020 WEEK1 Web Writeup

hgame 又开始了，寒假正好有时间，就参与一下了

Author: Th0tH

Cosmos 的博客

进入题目，提示github和版本管理工具，可以想到是Git泄露。尝试访问.git/config

可以得到github地址。

访问github地址，查看commit记录，得到flag

接 头 霸 王

点开题目，提示我需要访问，通过抓包解决

flag: hgame{W0w!Your_heads_@re_s0_many!}

Code World

点进题目，发现是403，查看代码发现提示

发现302跳转，用bp抓包截跳转的包，发现405

修改请求方式为POST，提示“人鸡验证”，提交参数为两个数的相加要为10，参数为a。输入a=1+9显示再想想

考虑是传递参数中的转义问题，url的+号会被转义为空格，因此使用urlencode的+号%2b

a=1%2b9

得到flag: hgame{C0d3_1s_s0_S@_sO_C0ol!}

🐔尼泰玫

不说了，又黑我坤坤，可恶。

一个js的游戏关，那么很容易就想到修改最后传递的参数。

游戏提示得到3W分就能获得flag，那怎么行了，要把坤坤累死的。

所以选择bp抓最后传递参数的包，将score修改为30000，得到flag。

flag: hgame{j4vASc1pt_w1ll_tel1_y0u_someth1n9_u5efu1?!}
